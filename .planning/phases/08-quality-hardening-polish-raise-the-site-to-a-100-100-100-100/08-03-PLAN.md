**Step 1 -- Create `lighthouserc.json` at project root (per D-01, D-02):**

```json
{
  "ci": {
    "collect": {
      "staticDistDir": "./dist",
      "url": ["/index.html", "/now/index.html"]
    },
    "assert": {
      "assertions": {
        "categories:performance": ["error", { "minScore": 1 }],
        "categories:accessibility": ["error", { "minScore": 1 }],
        "categories:best-practices": ["error", { "minScore": 1 }],
        "categories:seo": ["error", { "minScore": 1 }]
      }
    },
    "upload": {
      "target": "temporary-public-storage"
    }
  }
}
```

Key: `staticDistDir` mode means lhci starts its own static server from `dist/` -- no need for a running dev/preview server. This eliminates network variance for Performance scores.

**Step 2 -- Create `.github/workflows/ci.yml` (per D-03, D-04, D-05):**

First ensure the directory exists: `mkdir -p .github/workflows`

```yaml
name: CI

on:
  push:
    branches: [master]
  pull_request:

jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: "22"
          cache: "npm"
      - run: npm ci
      - run: npm run build
      - run: npx astro check
      - uses: actions/upload-artifact@v4
        with:
          name: dist
          path: dist/
          retention-days: 1

  unit-tests:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: "22"
          cache: "npm"
      - run: npm ci
      - run: npm test

  e2e:
    needs: build
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: "22"
          cache: "npm"
      - run: npm ci
      - run: npx playwright install --with-deps chromium
      - uses: actions/download-artifact@v4
        with:
          name: dist
          path: dist
      - run: npx playwright test
        env:
          CI: "true"

  lighthouse:
    needs: build
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: "22"
          cache: "npm"
      - run: npm ci
      - uses: actions/download-artifact@v4
        with:
          name: dist
          path: dist
      - run: npx lhci autorun --config=lighthouserc.json
        env:
          LHCI_GITHUB_APP_TOKEN: ${{ secrets.LHCI_GITHUB_APP_TOKEN }}
```

Architecture notes (per D-05 parallel jobs):
- `build` job: builds the site, runs `astro check` (typecheck), uploads `dist/` artifact. This is the shared dependency.
- `unit-tests` job: no dependency on build -- runs `vitest run` independently in parallel with build.
- `e2e` job: `needs: build` to download the `dist/` artifact. Playwright config's `webServer` runs `npm run preview` which serves the built dist.
- `lighthouse` job: `needs: build` to download `dist/`. Uses `npx lhci autorun` with `staticDistDir` (no webServer needed -- lhci handles it).
- Net parallelism: `build` and `unit-tests` start immediately. Once `build` finishes, `e2e` and `lighthouse` start in parallel.
- `LHCI_GITHUB_APP_TOKEN` is optional -- if not set, lhci still runs and asserts, it just doesn't post results to GitHub Checks. The user can configure it later.
- CF Analytics beacon: `astro build` sets `PROD=true`, so the beacon IS in `dist/`. In `staticDistDir` mode the beacon script tag exists but doesn't execute (no real browser context for the fetch). If it causes Best Practices issues, the user can set `PUBLIC_CF_ANALYTICS_TOKEN=""` in CI env, but this is unlikely.
