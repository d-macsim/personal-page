## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px (1 unit) | Icon gaps, button icon-to-label spacing |
| sm | 8px (2 units) | Inline padding, tag/badge spacing |
| md | 16px (4 units) | Default element spacing, button padding |
| lg | 24px (6 units) | Card padding, highlight card inner spacing |
| xl | 32px (8 units) | Between stacked hero elements (name → title → tagline → CTAs) |
| 2xl | 48px (12 units) | Gap between hero and about section |
| 3xl | 64px (16 units) | Top/bottom padding for major sections (hero, about) |

Exceptions:
- Hero section height: `100dvh` — full viewport, not a spacing token value
- Prose max-width: 680px (42.5rem) — established in Phase 1 global.css
- CTA button min-width: 160px — ensures both buttons are visually balanced at the same width
- Touch targets (mobile): minimum 44px height on CTA buttons

Source: global.css (`--spacing: 0.25rem` base unit), 02-CONTEXT.md (D-01, D-05), Phase 1 defaults

## Typography

All values use Inter Variable. Four semantic roles from global.css — no new sizes introduced in Phase 2.

| Role | Size (CSS token) | Rendered Range | Weight | Line Height | Usage in Phase 2 |
|------|-----------------|----------------|--------|-------------|------------------|
| Display | `--font-size-display` = `clamp(2.5rem, 5vw, 4rem)` | 40px–64px | 700 (bold) | 1.1 | Hero name (h1) |
| Heading | `--font-size-heading` = `clamp(1.25rem, 3vw, 2.5rem)` | 20px–40px | 600 (semibold) | 1.2 | Hero title, "About" section heading (h2), highlight card labels (h3) |
| Body | `--font-size-body` = `clamp(1rem, 2vw, 1.125rem)` | 16px–18px | 400 (regular) | 1.7 desktop / 1.6 mobile | Hero tagline, bio paragraphs |
| Label | `--font-size-label` = `0.875rem` (14px fixed) | 14px | 400 (regular) | 1.5 | Highlight card supporting text, meta info |

Weights in use: 400 (regular) and 600 (semibold) for most content; 700 (bold) reserved exclusively for the h1 display name.

Source: global.css (all four tokens), 01-CONTEXT.md (D-12: three weights 400/600/700, D-13: 18px body / 1.7 line-height)

| Text | `#e4e4e7` (`--color-text`) | `#18181b` | All body and heading text |
| Muted text | `#71717a` (`--color-text-muted`) | `#71717a` | Hero professional title, label text, card supporting copy |
| Border | `#27272a` (`--color-border`) | `#e4e4e7` | Highlight card border, ghost button border |
| Destructive | N/A | N/A | No destructive actions in Phase 2 |

**Accent reserved for (Phase 2 specific):**
1. "View my work" primary CTA button — filled indigo background
2. Radial glow behind hero name — soft indigo radial gradient (`--color-accent-primary` at 15–20% opacity)
3. Glow pulse animation — same indigo, opacity oscillates 10%–25% over 3–4s cycle
4. "Download CV" ghost button — amber (`--color-accent-secondary`) border and label text
5. Focus ring on both CTA buttons — 2px `--color-accent-primary` outline for keyboard navigation

Glow specification:
- CSS: `radial-gradient(ellipse 60% 40% at 50% 0%, rgba(99,102,241,0.20) 0%, transparent 70%)`
- Positioned behind h1 name text, centered horizontally, anchored to top of hero content block
- Pulse: keyframe animation alternating opacity from 0.12 to 0.22 over 4s `ease-in-out infinite`

### Animation Sequence (Motion v12, React island)

All animations respect `prefers-reduced-motion` — if active, render all elements at full opacity/position immediately.

| Step | Element | Delay | Animation |
|------|---------|-------|-----------|
| 1 | Hero name (h1) | 0ms | Fade in + translateY from +20px to 0 over 600ms, easing: `[0.16, 1, 0.3, 1]` |
| 2 | Professional title | +150ms | Same fade-up |
| 3 | Tagline | +300ms | Same fade-up |
| 4 | CTA buttons | +450ms | Same fade-up |
| 5 | Scroll indicator | +600ms | Same fade-up, then static |
| Continuous | Radial glow | starts at 0ms | Opacity pulse: 0.12 → 0.22 → 0.12, 4000ms ease-in-out infinite |

Motion v12 API: `import { motion } from "motion/react"`. Use `initial`, `animate`, `transition` props. Wrap each staggered item in `<motion.div>`.

- Background: `--color-accent-primary` (#6366f1 dark / #4f46e5 light)

**Ghost — "Download CV":**

Both buttons: `border-radius: 6px`, `padding: 12px 24px` (3 units × 6 units), `font-weight: 600`, `font-size: --font-size-label` (14px).

## SEO Metadata Contract

### Open Graph Tags (added to Head.astro)

```
og:title    → "Dragos Macsim — AI Specialist & Product Builder"
og:description → "Dragos Macsim — AI specialist who builds products. Explore projects, experience, and download CV."
og:type     → "website"
og:url      → canonical site URL
og:image    → "/og-image.png" (static 1200×630 PNG, designed file)
og:image:width  → 1200
og:image:height → 630
twitter:card    → "summary_large_image"
twitter:title   → same as og:title
twitter:description → same as og:description
```

OG image approach: static PNG at `/public/og-image.png`. Design: dark background (`#0a0a0f`), centered name in Inter 700 at ~48px white, subtitle below in 400 at ~24px muted (#71717a), soft indigo radial glow behind text. Create once manually — no build-time generation.

### JSON-LD Structured Data

Schema: `Person`. Fields: `name`, `jobTitle`, `url`, `sameAs` (LinkedIn, GitHub). Added as `<script type="application/ld+json">` inside Head.astro. No `image` field until real photo is available.

### Heading Hierarchy

- `h1`: Hero name — "Dragos Macsim" (one per page)
- `h2`: Section titles — "About" (one per section)
- `h3`: Highlight card labels — "Current Role", "Education"

## Accessibility Contract

- `prefers-reduced-motion`: if active, all Motion animations are skipped — all elements render at final state immediately (source: 02-CONTEXT.md D-11)
- Keyboard navigation: both CTA buttons receive visible focus ring (2px outline, `--color-accent-primary`)
- Scroll indicator chevron: `aria-label="Scroll to explore"`, `role="img"` on SVG
- Color contrast: all text on `--color-base` background meets WCAG AA (4.5:1 for body, 3:1 for large display text)
- Semantic HTML: `<section>` elements with `aria-label` for hero and about; `<h1>` only once per page
- OG image alt: not applicable (decorative social share image)
