## Spacing Scale

Declared values (all multiples of 4 — using Tailwind v4 `--spacing: 0.25rem` base unit):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px (1) | Icon gaps, inline offsets |
| sm | 8px (2) | Tight element separation |
| md | 16px (4) | Spine position (`left-4`), dot position (`left-4`) — THIS IS THE ALIGNMENT FIX |
| lg | 24px (6) | — |
| xl | 32px (8) | — |
| 2xl | 48px (12) | Entry bottom padding desktop (`pb-12`), content indent (`pl-12`) |
| 3xl | 64px (16) | Section vertical padding (`py-16` mobile), column gap (`gap-16`) |
| section | 96px (24) | Section vertical padding desktop (`py-24`) |

**Critical spacing constraint — dot alignment:**
- Spine bar: `left-4` = 16px from container left edge
- Dot marker: `left-4 -translate-x-1/2` — left-4 (16px) minus 50% of dot width (w-2.5 = 10px, so 5px) = dot centre at 16px. This matches the spine exactly.
- The previous value `left-3` (12px) placed the dot centre at 12px, misaligned 4px left of the 16px spine.
- Fix: change `left-3` to `left-4` on the dot element in the new `TimelineColumn.astro`.

**Column gap:**
The two-column grid uses `gap-12` (48px) at `md` breakpoint and above. This maintains visual breathing room between the Experience and Education columns without crowding either timeline's content.

Exceptions: none beyond the spine/dot coordinate described above.

## Component Inventory

### New: `TimelineColumn.astro`

Replaces `ExperienceTimeline.astro`. Generic — no CV-domain knowledge.

**Props interface:**
```typescript
interface TimelineEntry {
  heading: string;      // role.title or entry.degree
  subheading: string;   // role.company or entry.institution
  dateRange: string;
  body: string[];       // role.achievements or mapped education detail/modules
}

interface Props {
  title: string;           // "Experience" or "Education"
  entries: readonly TimelineEntry[];
}
```

**Rendered structure:**
```
div.relative.reveal-stagger
  h3 {title}
  div.absolute.left-4.top-0.bottom-0.w-px [spine — hidden on mobile]
  {entries.map → div.relative.pl-0.sm:pl-12.pb-8.sm:pb-12}
    div.absolute.left-4.top-2.w-2.5.h-2.5.rounded-full.-translate-x-1/2 [dot — hidden on mobile]
    span.label [dateRange — uppercase, tracking-widest, --color-text-muted]
    h3.font-semibold [heading · subheading]
    ul.list-disc.list-inside > li [body items — text-sm, --color-text-muted]
```

**Critical classes:**
- Spine: `absolute left-4 top-0 bottom-0 w-px hidden sm:block` + `background-color: var(--color-border)`
- Dot: `absolute left-4 top-2 w-2.5 h-2.5 rounded-full -translate-x-1/2 hidden sm:block` + `background-color: var(--color-accent-primary)`
- Entry wrapper: `relative pl-0 sm:pl-12 pb-8 sm:pb-12`
- `reveal-stagger` on the root div — enables stagger animation per column independently

### Modified: `CVSection.astro`

**Layout wrapper replacing single `max-w-2xl` column:**
```astro
<div class="grid grid-cols-1 md:grid-cols-2 gap-12">
  <TimelineColumn title="Experience" entries={experienceEntries} />
  <TimelineColumn title="Education" entries={educationEntries} />
</div>
```

**Data mapping (done at call site in CVSection, not inside TimelineColumn):**
- `roles` → `TimelineEntry[]` via `{ heading: r.title, subheading: r.company, dateRange: r.dateRange, body: r.achievements }`
- `education` → `TimelineEntry[]` via `{ heading: e.degree, subheading: e.institution, dateRange: e.dateRange, body: e.modules ? [e.detail, ...] : [e.detail] }`

**Section h2 update:**
- Before: `<h2>Experience</h2>`
- After: `<h2>Experience & Education</h2>` (accurately labels both columns)

**Skills and download button:** remain full-width below the two-column grid — no layout change.

## Responsive Behaviour

| Breakpoint | Layout | Notes |
|------------|--------|-------|
| < 768px (< md) | Single column, stacked — Experience first, Education second | Identical to current behaviour; no regression |
| ≥ 768px (md+) | Two equal-width columns side by side via CSS Grid | Native CSS Grid ensures equal heights without JS |
| All widths | Timeline spine and dots hidden below `sm` (640px) | Existing `hidden sm:block` classes preserved |

Mobile first — `grid-cols-1` is the default, `md:grid-cols-2` applies at 768px+.

## Animation Contract

No new animations introduced. Existing `reveal-stagger` class is preserved on each `TimelineColumn` root div. This ensures each column's entries stagger independently using the Phase 5 CSS scroll-driven animation rules in `global.css`.

Phase 5 decision: stagger via `animation-range` nth-child offsets (not `animation-delay`). Unchanged.

`@supports (animation-timeline: view())` gate in `global.css` ensures graceful degradation on non-supporting browsers.

## Accessibility Contract

| Concern | Requirement |
|---------|-------------|
| Heading hierarchy | Section h2 → column h3 → entry h3 (use `text-sm font-semibold` for entry heading, not a semantic h4, to avoid four-level nesting in a flat list) |
