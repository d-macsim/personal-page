## Spacing Scale

Declared values — all multiples of 4 (Tailwind v4 `--spacing: 0.25rem` base, so every integer utility is a 4px step):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px (1 unit) | Icon gaps, inline pill padding |
| sm | 8px (2 units) | Badge internal padding, compact element gaps |
| md | 16px (4 units) | Default element spacing, nav link gaps |
| lg | 24px (6 units) | Card internal padding, stacked element rhythm |
| xl | 32px (8 units) | Section-internal layout gaps |
| 2xl | 48px (12 units) | Section vertical padding (mobile: `py-16`) |
| 3xl | 64px (16 units) | Section vertical padding (desktop: `py-24`) |

**Source:** Phase 1 design system, global.css `--spacing: 0.25rem`; section padding pattern verified in CVSection.astro and AboutSection.astro.

Exceptions:
- Nav bar height: 56px (h-14 = 3.5rem) — deviates from pure 8-pt scale to match standard sticky nav ergonomics
- Touch targets for hamburger button and nav icon links: minimum 44px — accessibility requirement (WCAG 2.5.5)
- Device frame placeholder: 240px wide × 480px tall — fixed art dimensions, not a spacing token

## Interaction Contracts

### Nav — Active Section Highlighting (D-08)

| State | Visual Treatment |
|-------|-----------------|
| Default nav link | `color: var(--color-text-muted)`, weight 500, no underline |
| Hover nav link | `color: var(--color-text)`, transition 200ms ease |
| Active nav link (`data-active="true"`) | `color: var(--color-accent-primary)`, 2px underline offset 2px |
| Nav backdrop | `background: rgba(10,10,15,0.8)` (dark) / `rgba(250,250,250,0.8)` (light), `backdrop-blur-sm`, `border-bottom: 1px solid var(--color-border)` |

Intersection Observer `rootMargin: '-40% 0px -55% 0px'` — activates at viewport center band, prevents flicker at section boundaries. Source: RESEARCH.md Pattern 4.

### Nav — Hamburger Menu (D-07)

| State | Visual Treatment |
|-------|-----------------|
| Closed | `<button>` with 3-bar SVG icon, `aria-expanded="false"`, `aria-label="Open navigation menu"` |
| Open | X icon (or same icon), `aria-expanded="true"`, mobile dropdown visible |
| Mobile menu | `background: var(--color-base)`, full width, `border-bottom: 1px solid var(--color-border)`, stacked nav links `py-3 px-4` each |
| Menu toggle animation | CSS `transition` on `max-height` or `hidden` class toggle — no Motion (Phase 5 only) |

### Project Card CTA Button (D-04)

| State | Visual Treatment |
|-------|-----------------|
| Default | `background: var(--color-accent-primary)`, `color: white`, `rounded-lg`, `px-6 py-3`, `font-weight: 600` |
| Hover | `background: var(--color-accent-primary-hover)`, `transition-colors duration-200` |
| Active/pressed | `background: var(--color-accent-primary-press)` |

### Contact Icon Links (D-13)

| State | Visual Treatment |
|-------|-----------------|
| Default | `border: 1px solid var(--color-border)`, `background: var(--color-surface)`, `rounded-lg`, `px-6 py-4`, inline-flex with icon + label |
| Hover | `border-color: var(--color-accent-primary)`, `color: var(--color-accent-primary)`, `transition-colors duration-200` |

### Smooth Scroll (D-09)

Implementation: `element.scrollIntoView({ behavior: 'smooth' })` per nav link click handler — matches HeroSection.tsx existing pattern (verified line 76-79). Do NOT use global CSS `scroll-behavior: smooth` (conflict with React hero).

## Layout Contracts

### Mobile (< 768px)

- Nav: logo/name left + ThemeToggle + hamburger right; section links hidden in dropdown
- Project card: `flex-col` — device frame full width (max-w-[240px] mx-auto), details below
- Contact links: `flex-col gap-4` stacked vertically, full width
- Section padding: `py-16` (64px top/bottom)

### Desktop (≥ 768px)

- Nav: logo/name left + section links center-right + ThemeToggle right; hamburger hidden
- Project card: `md:flex-row gap-12` — device frame left (flex-shrink-0 w-[240px]), details right
- Contact links: `flex-row gap-6` horizontal row, center-aligned
- Section padding: `md:py-24` (96px top/bottom)

### Nav positioning

Fixed sticky: `position: fixed; top: 0; left: 0; right: 0; z-index: 50`. Page content needs `pt-14` on the layout wrapper to prevent overlap.

## Accessibility Contracts

| Requirement | Implementation |
|-------------|----------------|
| Hamburger button | `<button>` element (not div), `aria-label="Open navigation menu"`, `aria-expanded` toggled by script |
| Mobile menu visibility | Use `hidden` class (removes from DOM flow) — not `opacity-0` or `visibility: hidden` |
| External links | `target="_blank" rel="noopener noreferrer"` on all LinkedIn and GitHub links (D-12) |
| Contact email | `href="mailto:..."` — no JS-encoded obfuscation |
| Nav landmarks | `<nav aria-label="Main navigation">` wrapper |
| Section landmarks | `<section aria-label="Projects" id="projects">` and `<section aria-label="Contact" id="contact">` |
| Touch targets | Minimum 44px × 44px for hamburger button and contact icon links |
| Device frame placeholder | `aria-label="mytai app preview"`, gradient overlay `aria-hidden="true"` |
| Focus management | All interactive elements keyboard-reachable; active state visible at `outline: 2px solid var(--color-accent-primary)` |
| Reduced motion | Nav toggle animation uses CSS `hidden` class (no animation) — inherently respects reduced motion |
| Color contrast | `var(--color-text)` on `var(--color-base)`: #e4e4e7 on #0a0a0f = ~15:1 (exceeds WCAG AA 4.5:1) |
