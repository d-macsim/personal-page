## Spacing Scale

Declared values — 4px base unit, established in Phase 1 `global.css` (`--spacing: 0.25rem`):

| Token | Value | Usage in Phase 3 |
|-------|-------|-----------------|
| xs | 4px (`gap-1`) | Skill badge inner padding horizontal supplement |
| sm | 8px (`gap-2`) | Gap between skill badges in flex-wrap row |
| md | 16px (`gap-4`) | Timeline entry internal content spacing |
| lg | 24px (`mb-6`, `gap-6`) | Space between skill category groups |
| xl | 32px (`pb-8`) | Vertical gap between timeline entries (mobile) |
| 2xl | 48px (`pb-12`) | Vertical gap between timeline entries (desktop) |
| 3xl | 64px (`mt-16`) | Gap between timeline block and skills block |

Section padding follows established pattern: `py-16 md:py-24` (64px / 96px).

Exceptions:
- Timeline dot marker: 10px diameter (`w-2.5 h-2.5`) — off-scale intentionally for visual weight at the connector point
- Touch targets on download button: minimum 44px height (`py-3` = 12px vertical padding + 20px line-height = 44px effective height) — meets WCAG 2.5.8 minimum target size

## Typography

All sizes use existing CSS custom properties from `src/styles/global.css`. No new variables.

| Role | Size token | Computed range | Weight | Line Height | Usage |
|------|-----------|----------------|--------|-------------|-------|
| Display | `--font-size-display` | `clamp(2.5rem, 5vw, 4rem)` (40–64px) | 700 | 1.1 | Not used in Phase 3 |
| Heading | `--font-size-heading` | `clamp(1.25rem, 3vw, 2.5rem)` (20–40px) | 600 | 1.2 | Section `<h2>` ("Experience", "Skills") |

**Weights in use:** 400 (regular) and 600 (semibold) only — matching established Phase 1 constraint.

**Source:** `src/styles/global.css` @theme block — no changes needed.

## Color

| Role | Dark value | Light value | Usage in Phase 3 |
|------|-----------|-------------|-----------------|
| Dominant (60%) — `--color-base` | `#0a0a0f` | `#fafafa` | Page background (inherited, no new usage) |

| Border — `--color-border` | `#27272a` | `#e4e4e7` | Timeline spine line, skill badge border |
| Accent (10%) — `--color-accent-primary` | `#6366f1` | `#4f46e5` | Timeline dot markers, download CTA button fill |
| Accent hover — `--color-accent-primary-hover` | `#818cf8` | `#6366f1` | Download button hover state |
| Accent press — `--color-accent-primary-press` | `#4f46e5` | `#3730a3` | Download button active/press state |
| Text — `--color-text` | `#e4e4e7` | `#18181b` | Role titles, company names in timeline, skill badge text |
| Text muted — `--color-text-muted` | `#71717a` | `#71717a` | Date ranges, achievement bullets, education details, category labels |

**60/30/10 split:**
- 60%: `--color-base` — page background
- 30%: `--color-surface` + `--color-border` — skill badge backgrounds, timeline spine
- 10%: `--color-accent-primary` — timeline dots, download button only

**Accent reserved for:**
1. Timeline dot markers at each entry (experience + education)
2. The CV download CTA button fill and its hover/press states
3. Nothing else — do not apply accent to heading text, skill badges, or section borders

**No destructive color needed** — Phase 3 contains no destructive actions.

**Source:** `src/styles/global.css` Phase 1 tokens — no new color values introduced.

## Component Inventory

New components to build in Phase 3. All are static Astro components (no React island).

| Component | File | Pattern |
|-----------|------|---------|
| CVSection | `src/components/CVSection.astro` | Section wrapper — mirrors `AboutSection.astro` structure: `<section aria-label="Experience and CV" id="experience" class="py-16 md:py-24">` |
| ExperienceTimeline | `src/components/ExperienceTimeline.astro` | Vertical CSS timeline — spine line + dot markers + two-column layout collapsing to single-column on mobile |
| SkillsGrid | `src/components/SkillsGrid.astro` | Flex-wrap badge rows grouped by category |
| CVDownloadButton | `src/components/CVDownloadButton.astro` | Native `<a download>` anchor styled as solid indigo primary button |

**Data source:** `src/data/cv.ts` — TypeScript file with typed exports for `roles`, `education`, `skillCategories`. All text content lives here; components only render structure.

### Timeline Layout (ExperienceTimeline)

**Desktop (sm+):** Two-column layout — date range in a fixed-width left column (≈96px), role content in the right column. Left border spine runs the full height of the container. Dot marker centered on the spine at each entry top.

**Mobile (<640px):** Single column. Date range rendered as a small `<span>` above the role title (not hidden — always visible). Spine line hidden on mobile (`hidden sm:block`). Dot markers hidden on mobile (line is hidden, dots have no reference).

**Spine geometry:**
- Spine: `absolute left-4 top-0 bottom-0 w-px` with `background-color: var(--color-border)`
- Dot: `absolute left-3 top-2 w-2.5 h-2.5 rounded-full -translate-x-1/2` with `background-color: var(--color-accent-primary)`
- Content indent: `pl-12` on each entry to clear the spine + dot

**Education separator:** A styled `<div>` (not a heading element) with text "Education" at label size, uppercase, tracking-widest, `--color-text-muted`. Uses `mt-8 mb-4` spacing. Appears between last work entry and first education entry within the same timeline container.

**Section `id`:** `id="experience"` on the outer `<section>` for future Phase 4/5 anchor navigation.

### Skill Badges (SkillsGrid)

- Category heading: `<h3>` at label size (14px), weight 600, uppercase, tracking-widest, `--color-text-muted`
- Badge: `<span>` — `px-3 py-1 rounded-full text-sm font-medium` — `background-color: var(--color-surface)`, `border: 1px solid var(--color-border)`, `color: var(--color-text)`
- No hover state on badges — they are display-only, not interactive
- Category gap: `mb-6` between groups

**Skill category groupings (Claude's discretion — D-09 area):**
| Category | Skills |
|----------|--------|
| Languages & Data | Python, R, SQL, Excel |
| AI & ML | RAG, LLM Evaluation, Deep Learning, BeautifulSoup, Selenium |
| Tools & Infrastructure | Tableau, Docker, Ubuntu Server, Git |

### Download Button (CVDownloadButton)

- Element: `<a href="/Dragos Macsim CV 2026.pdf" download="Dragos Macsim CV 2026.pdf">`
- Style: solid indigo primary — fills with `--color-accent-primary`, text `white`, `font-weight: 600`
- Size: `inline-flex items-center gap-2 px-6 py-3 rounded-lg` — effective height ≥ 44px
- Icon: inline SVG download arrow (↓) at 16×16px to the left of the label — adds scannability without library dependency
- Hover: `background-color: var(--color-accent-primary-hover)` via `transition-colors duration-200`
- Press: `background-color: var(--color-accent-primary-press)`
- Placement: `mt-12` below the SkillsGrid, left-aligned (not centered)

**Rationale for solid over ghost (D-09):** At the end of the experience section the download is the primary conversion action. Ghost style is reserved for the hero where "View my work" is the primary CTA and download is secondary. Two solid buttons at different page positions (hero + CV section end) is intentional and acceptable.

## Interaction Contract

| Interaction | Element | Behavior |
|-------------|---------|----------|
| CV download | `<a download>` | Browser save dialog triggered; file saves as "Dragos Macsim CV 2026.pdf" |
| Download button hover | `.cv-download-btn:hover` | Color transitions to `--color-accent-primary-hover` over 200ms |
| Download button press | `.cv-download-btn:active` | Color transitions to `--color-accent-primary-press` |
| Dark/light theme switch | inherited | `transition-colors duration-300` on surface and border elements |
| Timeline entries | none | Static — no hover, no expand/collapse, no interaction |
| Skill badges | none | Static display only — no hover, no click, no tooltip |

**No JavaScript required** for any interaction in this phase. The download is native HTML. Hover/press states are pure CSS.

**Phase 5 note:** Scroll-reveal animations will be added to this section in Phase 5 (Motion v12). Build the section without any animation hooks or Motion imports now — Phase 5 will layer these in without requiring component restructuring.

## Responsive Behavior

| Breakpoint | Layout behavior |
|------------|----------------|
| Mobile `<640px` | Single-column. Timeline: date as `<span>` above role title, spine hidden, dots hidden. Skills: badge flex-wrap unchanged. Download button: full-width block. |
| Tablet `≥640px` | Two-column timeline: fixed 96px date column + content. Spine and dots visible. Download button: inline-block (auto width). |
| Desktop `≥1280px` | Same as tablet. Section max-width capped at `max-w-2xl` (672px) for prose readability. Download button: inline-block. |

**Section max-width:** `max-w-2xl` (42rem / 672px) on the content container inside CVSection — matching the existing `prose` max-width pattern from AboutSection.

## Accessibility Contract

- `<section aria-label="Experience and CV" id="experience">` — landmark with descriptive label
- `<h2>` for "Experience" and "Skills" — peer section titles, consistent with established heading hierarchy
- Achievement bullets in `<ul>` / `<li>` — semantic list markup
- Download `<a>` with `aria-label="Download Dragos Macsim CV as PDF"` — describes file type for screen reader users
- Skill badges as `<span>` — decorative, non-interactive; no `role` or `tabindex` needed
- Color contrast: `--color-text-muted` (#71717a) on dark base (#0a0a0f) — 4.7:1 — passes WCAG AA for small text
- Color contrast: white on `--color-accent-primary` (#6366f1) — 4.54:1 — passes WCAG AA for button text
- `prefers-reduced-motion`: no animations in Phase 3 — full compliance by default
