      // Smooth scroll for hash anchors only — real routes (e.g. /now) navigate normally.
      // Without this guard, the handler would preventDefault() on /now clicks and the page would not load.
      const allNavLinks = document.querySelectorAll('[data-nav-link]');
      allNavLinks.forEach((link) => {
        link.addEventListener('click', (e) => {
          const href = link.getAttribute('href');
          if (!href || !href.startsWith('#')) return; // let real pages navigate normally
          e.preventDefault();
          const target = document.querySelector(href);
          target?.scrollIntoView({ behavior: 'smooth' });
        });
      });

      // Mark the /now link active when we're on the /now route.
      // Runs once at script load; the IntersectionObserver below only touches section[id] anchors,

The closing `</body>` is on line 133 (after the `</script>` that ends the inline behaviours, and after the `<style is:global>` block further down — note the `<style>` block is actually AFTER `</body>` which is valid Astro but unusual; double-check the file). Add the beacon snippet as the LAST child inside `<body>`, immediately after the closing `</script>` of the inline behaviour block and BEFORE `</body>`.

Insert (literally, at the correct location):

```astro
    {import.meta.env.PROD && cfAnalyticsToken && (
      <script
        is:inline
        defer
        src="https://static.cloudflareinsights.com/beacon.min.js"
        data-cf-beacon={`{"token": "${cfAnalyticsToken}"}`}
      />
    )}
```

Rationale per RESEARCH.md (lines 196-212) and UI-SPEC (lines 316-323):
- `import.meta.env.PROD` is `true` only in production builds (`astro build`), `false` in dev (`astro dev`). Prevents polluting real analytics with local traffic.
- `cfAnalyticsToken && ...` extra guard: if the env var is unset at build time, the beacon is NOT injected (fail-closed). This means a build without the token still succeeds but emits no analytics — acceptable default per A1 in RESEARCH.md assumptions.
- `is:inline` prevents Astro from bundling/processing the third-party script.
- `defer` lets the browser parse it after HTML completes (recommended per CF docs).
- The `data-cf-beacon` attribute uses a backtick template with the token interpolated at build time — Astro resolves `import.meta.env.PUBLIC_CF_ANALYTICS_TOKEN` statically.

CRITICAL: the token is PUBLIC (prefixed `PUBLIC_` so Astro exposes it to client code). This is intentional — CF Analytics tokens are site identifiers, not secrets. See RESEARCH.md line 196 and env.d.ts comment from Plan 01.
